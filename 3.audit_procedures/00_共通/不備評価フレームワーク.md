# 不備評価フレームワーク

**用途**: 運用状況評価で発見された例外事象の不備認定と影響度評価

---

## 1. 不備認定の判断フロー

```
統制テストで例外発見
  ↓
 例外の性質調査
  ↓
┌─────────────┬─────────────┐
│ 軽微な逸脱    │ 明確な不備     │
│ (業務影響なし) │ (統制失敗)     │
└─────────────┴─────────────┘
       ↓              ↓
  許容可能       不備認定
                    ↓
              影響度評価
                    ↓
  ┌─────────┬─────────┬─────────┐
  │ 軽微     │ 重要の可能性 │ 重要       │
  └─────────┴─────────┴─────────┘
       ↓           ↓           ↓
    是正記録    是正+開示検討  開示必須
```

## 2. 不備の種別定義

### 2-1. 不備の段階
| 種別 | 英名 | 定義 |
|------|------|------|
| **軽微な不備** | Deficiency | 単独または組合せにおいて、財務報告に重要な影響を与える可能性が低い |
| **重要な不備の可能性** | Significant Deficiency | 単独または組合せにおいて、財務報告に一定の影響を与える可能性がある |
| **重要な不備** | Material Weakness | 単独または組合せにおいて、財務報告に重要な虚偽記載をもたらす可能性が合理的に存在する |

### 2-2. 判定要素

#### 【金額的重要性】
想定誤謬金額を試算：
- 想定誤謬金額 = 不合格率 × 母集団金額
- 例：発注承認25件中3件不合格（12%）、年間発注総額 20億円
  → 想定誤謬金額 ≒ 20億円 × 12% = 2.4億円

判定基準（一般的な閾値）：
- 連結売上高の 5% 超：**重要**
- 連結売上高の 0.5% ～ 5%：**重要の可能性**
- 連結売上高の 0.5% 未満：**軽微**

当社（連結売上380億円）の場合：
- 19億円超：**重要**
- 1.9億円～19億円：**重要の可能性**
- 1.9億円未満：**軽微**

#### 【質的重要性】
金額だけでは判断できない要素を評価：
- **不正に関連するか**（関連する場合は金額が小さくても重要性増）
- **職務分掌違反を伴うか**
- **繰り返し発生しているか**
- **経営陣の関与があるか**
- **開示規制に関連するか**

## 4. 是正措置の要件

### 4-1. 重要な不備の場合
- **開示義務**：内部統制報告書に「開示すべき重要な不備」として記載
- **是正計画**：期末後90日以内に是正完了させる計画を策定
- **期末監査対応**：監査法人に速やかに報告・協議
- **取締役会報告**：次回取締役会で報告、対策を決議

### 4-2. 軽微な不備の場合
- 是正計画を策定
- 期末までに是正完了または継続的モニタリング
- 内部監査室への報告

## 5. 補完統制（Compensating Control）の検討

統制に不備があっても、以下の**補完統制**により最終的な財務諸表への影響を抑制できる場合：
- **重要性の評価を引き下げる**ことができる

### 5-1. 当社のケース

#### PLC-P-002 発注承認の補完統制
- ITGC-AC-004 特権ID管理（情シス部長レビューで不正検知可能）
- PLC-P-004 3-wayマッチング（実取引との整合性チェック）
- FCRP-001 月次決算チェック（異常値検知）
→ 補完統制により影響は「重要な不備の可能性」にとどまり、「重要な不備」には至らないと判断

#### ITGC-AC-003 退職者アカウントの補完統制
- ITGC-AC-004 特権ID管理月次レビュー
- PLC-P系の部門長承認（人手の統制）
→ システム利用状況のレビューで不正利用を検知可能だが、期間中の監視空白は埋まらない。補完統制を考慮しても「重要な不備」の認定は変わらず。
