# J-SOX 運用状況評価 サンプリング方法論

**作成者**: 内部監査室（E0081 長谷川 剛）
**作成日**: 2026/04/05
**改訂**: v1.0

---

## 1. 目的

本文書は、J-SOXの運用状況評価における統制テストに用いる**サンプル抽出方法**および**サンプル数**の考え方を定める。
日本公認会計士協会（JICPA）のガイドラインに準拠し、監査法人の実務慣行を踏まえた判定基準を採用する。

---

## 2. サンプル数の基本原則

統制の実施頻度に応じて、以下の標準サンプル数を採用する。

| 実施頻度 | 年間実施回数（目安） | 標準サンプル数 |
|---------|-------------------|--------------|
| 日次 | 250回以上 | **25件** |
| 週次 | 50回程度 | **15件** |
| 月次 | 12回 | **3-5件** |
| 四半期 | 4回 | **2件** |
| 半期 | 2回 | **2件（全数）** |
| 年次 | 1回 | **1件（全数）** |
| 随時（都度） | ― | **25件**（※母集団が少ない場合は全数） |

### 根拠

- 25件：信頼水準90%・許容誤差0%（または低許容誤差）での統制テストの標準サンプル数
- 低頻度統制（月次以下）：実施回数自体が少ないため、実施のすべてを評価対象にすることが可能

## 3. サンプル抽出方法

以下の2種のいずれかを採用する。

### 3-1. 系統抽出法（推奨）

母集団をN件とし、サンプル数nで**N÷n**の間隔で抽出する。

**手順**：
1. SAP等から母集団データ全件を取得（取得条件を明記）
2. 間隔 = 母集団件数 ÷ サンプル数（小数点以下切り上げ）
3. 開始位置を無作為に決定（1～間隔の範囲）
4. 開始位置から間隔ごとに抽出

**例**：年間受注3,000件からサンプル25件
- 間隔 = 3,000 ÷ 25 = 120件
- 開始位置 = 57（無作為）
- 抽出対象 = 57, 177, 297, 417, ... 番目

### 3-2. 無作為抽出法

乱数生成により母集団から無作為にn件を抽出。大規模な母集団で偏りを避けたい場合に使用。

---

## 4. 母集団の定義

### 4-1. 記載必須事項
- 母集団の出所（SAPトランザクションコード、Excelファイル名等）
- 抽出条件（対象期間、ステータス等）
- 抽出日時
- 抽出件数

### 4-2. 例

```
母集団：FY2025期間中（2025/4/1～2026/3/31）にSAP SD（販売）で登録された全受注
出所：SAP トランザクション VA05 （「受注伝票一覧」）
抽出条件：
  - 受注日: 2025/4/1 - 2026/3/31
  - 受注ステータス: 「完了」または「処理中」
  - キャンセル済を除く
抽出日時：2026/2/10 10:30 JST
抽出件数：3,247件
```

## 5. 評価判定基準（25件サンプルの場合）

| 不合格件数 | 判定 | 対応 |
|-----------|------|------|
| 0件 | **有効** | 運用状況有効として結論付け |
| 1件 | **追加テスト検討** | 例外分析を実施し、軽微な場合は許容、そうでない場合は追加25件テスト |
| 2件 | **追加テスト必須** | 追加25件のテストを実施し、再評価 |
| 3件以上 | **不備** | 不備として認定し、不備管理台帳に記録 |

**判断保留案件**：追加エビデンスを入手後に再判定。期末までに解消しない場合は不備として認定。

---

## 6. 不備の分類

### 6-1. 不備の種別
| 種別 | 定義 | 会計上の影響 |
|------|------|------------|
| **軽微な不備** | 経営者及び監査人の財務報告における判断に影響を与える可能性が低い | 開示不要 |
| **重要な不備の可能性** | 不備の影響度が軽微を超えるが重要までは至らない | 場合により開示 |
| **重要な不備** | 財務報告の重要な虚偽記載につながる可能性がある | 開示必須 |

### 6-2. 影響度評価の要素
- **金額的重要性**：不備により生じうる誤差金額
- **質的重要性**：関連する統制の性質、不正の可能性、影響範囲の広さ

---

## 7. 記録要件

### 7-1. サンプル一覧（Excel）
各サンプルについて以下を記録：
- サンプル番号（No.1～25）
- 取引ID / 取引日 / 金額 / 取引先
- 検証ポイント（複数）
- 判定結果（合格/不合格/保留）
- 不合格・保留の理由

### 7-2. テスト結果記録
- 総サンプル数
- 合格件数 / 不合格件数 / 保留件数
- 結論（有効/不備）
- 不備認定の場合：不備の性質・影響・是正計画
